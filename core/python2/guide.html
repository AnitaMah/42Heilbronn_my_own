Python Module 02: Garden Guardian

Data Engineering for Smart Agriculture. Цей модуль присвячений створенню стійких конвеєрів даних шляхом опанування обробки виключень[cite: 10, 17, 75].

Проєкт та концепції

Сучасне сільське господарство вимагає систем, які витончено обробляють неочікувані збої (наприклад, несправності датчиків або розриви мережі)[cite: 12, 13, 15, 98].

  • Resilient Design: Системи, що не виходять з ладу при помилках[cite: 15, 96].
  • Exception Handling: Використання try, except, finally, raise[cite: 17, 101, 102].
  • Custom Exceptions: Створення власних типів помилок[cite: 165].

Концепції обробки виключень

Розуміння того, як потік виконання змінюється при помилці, є критичним[cite: 98, 101].

Псевдокод та приклади

    // Приклад Вправа 0: Базова валідація
    try:
        temp = int(input_str) // Спроба конвертації
    except ValueError:
        print("Помилка: некоректні дані")

    // Приклад Вправа 4: Гарантоване очищення
    try:
        відкрити_систему_поливу()
        поливати()
    finally:
        закрити_систему_поливу() // Виконається ЗАВЖДИ

Ключові поради

  • raise: Використовуйте для порушення бізнес-логіки, якщо дані поза межами (наприклад, 0–40°C)[cite: 146].
  • Специфічність: Ловіть конкретні помилки (ValueError, TypeError), а не Exception, щоб уникнути приховування багів[cite: 103, 160].
  • Ієрархія: Створюйте власні класи через успадкування для кращого групування помилок[cite: 165, 171].

Загальні правила

  • Мова: Python 3.10+[cite: 88].
  • Стандарти: flake8, type hints[cite: 89, 90].
  • Оборонне програмування: Програми ніколи не повинні "падати"[cite: 96].
  • AI Policy: Використовуйте ШІ, але ви повинні повністю розуміти та обґрунтувати свій код під час захисту[cite: 34, 61].
